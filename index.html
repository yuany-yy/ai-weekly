Anthropic 内部人员怎么用 Claude：一个让 AI 持续汇报进度的 prompt

Thariq（Claude Code 团队）分享了同事 Suzanne 的做法：用一个专门的 prompt 让 Claude 每完成一步就汇报状态，而不是默默跑完再告诉你结果。这条推文获得 6000+ likes、350+ 转发，是本周 AI 工程师社区互动最高的内容。对每天用 Claude 跑复杂任务的人，这是一个立刻可用的工作流改进。Suzanne 还提到配合语音模式使用，反馈更自然。

在长任务开始前加一句「每完成一个步骤，先汇报进度再继续」，成本极低，但对掌控感的提升非常明显

独立开发者用 AI agents 一人维护多个产品的 6 条生存法则

Peter Yang 整理了有 25 年经验的独立开发者 Josh Shpigford 的方法论：从 day 1 就收费并砍掉无法自负盈亏的产品、用 git worktrees 并行开发多个功能互不干扰、用 GPT 审 Claude 的代码（GPT 每次能多找出 3-5 个 bug）、构建 /learnings skill 让 agent 从错误中积累规则。这套方法专为「一个人 + 多个 AI agents」的工作模式设计。

/learnings skill 的思路：每个开发阶段结束后让 AI 自己总结「哪些做法不要重复」，相当于给 agent 建了一个可积累的错误手册，越用越聪明
